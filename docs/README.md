# docs/ — Documentación del proyecto

Documentación de arquitectura, decisiones y material de trabajo. **No es fuente de
conocimiento del motor**: nada de lo que esté aquí es servible al usuario final.

## Contenido previsto

- `architecture.md` — arquitectura del sistema (borrador inicial incluido).
- Decisiones de diseño (formato ADR: contexto → decisión → consecuencias).
- Notas de trabajo y análisis. Si algún día contienen material jurídico, siguen sin ser
  servibles: la única vía hacia el usuario es `data/approved_criteria/` y `data/catalog/`
  (Reglas 1, 5 y 13).
